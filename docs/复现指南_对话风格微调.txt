一、总览：整个项目做了什么？
══════════════════════════════════════════════════════════════

  354 张微信长截图 → 结构化 JSON → ShareGPT 训练数据 → LoRA 微调 → 能模仿"阿狸"聊天的模型

  核心挑战：让 AI 学会一个人的聊天风格（撒娇、短回复、特定语气），
  而不是让它背下对话内容。

  ╔═══════════════════════════════════════════════════════════╗
  ║  关键发现：训练用泛用短 prompt，推理时注入人格化长 prompt  ║
  ║  训练 → "你正在微信上与朋友聊天。"（12 字）                ║
  ║  推理 → "你是阿狸，爱撒娇的女生，在和男朋友聊天。"（60 字）║
  ╚═══════════════════════════════════════════════════════════╝

三、数据从哪里来？
══════════════════════════════════════════════════════════════

  原始数据：354 张超长竖屏微信聊天截图（JPG），分散在 ~56 个分类文件夹中

  处理流程（已跑完，无需重跑）：
    1. split_v2.py        → 超长截图切成小段
    2. extract_all.py      → Kimi Vision API 提取文字 → sessions/*.json
    3. build_lora_data.py  → sessions → turns → 话题分割 → lora_data/

  中间产物 sessions/ 格式：
  ┌─────────────────────────────────────────────────────────┐
  │ {                                                        │
  │   "category": "0107道士下山",                             │
  │   "source_image": "20220107_122955.jpg",                 │
  │   "messages": [                                          │
  │     {"sender": "me", "text": "在干嘛", "time": "12:29"},  │
  │     {"sender": "them", "text": "想你", "time": "12:30"}   │
  │   ]                                                      │
  │ }                                                        │
  └─────────────────────────────────────────────────────────┘
  - "me" = 喵喵（男朋友，作为 human 输入）
  - "them" = 阿狸/河狸（女朋友，作为 gpt 学习目标）

  角色映射：
    me (喵喵)     → human  → 对话上文
    them (阿狸)   → gpt    → 模型要学习模仿的回复

四、如何构建训练数据？
══════════════════════════════════════════════════════════════

  前提：sessions/ 目录已有 354 个 JSON 文件。

  运行命令：
    cd E:\狸喵galgame
    ./galgame/venv/Scripts/python build_lora_data.py

  输出文件（均在 lora_data/ 下）：
    - train.json        ← 训练集
    - valid.json        ← 验证集
    - stats.json        ← 统计信息
    - dataset_info.json ← 保持不变（手动维护）

  构建管道四步：
    Step 1 — TXT emotion 修正：18 个 session 有手工黄金标注，覆盖 API 的粗糙标注
    Step 2 — Turn 构建：同一人的连续消息合并为一个 turn
    Step 3 — 话题分割：两次回复间隔 > 30 分钟则切分为新话题
    Step 4 — ShareGPT 输出：为每个 them 回复构造 (上文 human+gpt 交替) → 目标 gpt 的训练对

  关键参数（在 build_lora_data.py 顶部）：
    TIME_GAP_MINUTES = 30    # 话题分割阈值，超过此间隔切分
    CONTEXT_TURNS = 10       # 每条样本的上下文窗口（最近 10 轮对话）
    TRAIN_RATIO = 0.8        # 训练/验证集比例（按日期排序后切分）

  输出格式（ShareGPT）：
  ┌─────────────────────────────────────────────────────────┐
  │ [{                                                       │
  │   "source": "0107道士下山/20220107_122955.jpg",           │
  │   "conversations": [                                     │
  │     {"from": "system", "value": "你正在微信上与朋友聊天。"},│
  │     {"from": "human", "value": "在干嘛"},                 │
  │     {"from": "gpt",   "value": "想你呀"},                 │
  │     {"from": "human", "value": "今天天气不错"},           │
  │     {"from": "gpt",   "value": "那你带我出去玩"}  ← 目标  │
  │   ]                                                      │
  │ }]                                                       │
  └─────────────────────────────────────────────────────────┘

  ★ 重要规则：
    - system 后面第一条必须是 human（不能是 gpt）
    - LLaMA-Factory v0.9+ 会自动丢弃不满足此规则的样本
    - build_lora_data.py 已内置 first_me 过滤逻辑，自动跳过无效块

  统计（当前版本）：
    训练样本: 4,592 条
    验证样本: 1,148 条
    平均对话长度: ~8 轮

五、如何注册数据集？
══════════════════════════════════════════════════════════════

  dataset_info.json 内容（已生成，位于 lora_data/ 下）：
  ┌─────────────────────────────────────────────────────────┐
  │ {                                                        │
  │   "chat_style_train": {                                  │
  │     "file_name": "train.json",                           │
  │     "formatting": "sharegpt",                            │
  │     "columns": {"messages": "conversations"},            │
  │     "tags": {                                            │
  │       "role_tag": "from",                                │
  │       "content_tag": "value",                            │
  │       "user_tag": "human",                               │
  │       "assistant_tag": "gpt",                            │
  │       "system_tag": "system"                             │
  │     }                                                    │
  │   },                                                     │
  │   "chat_style_valid": { ... }                            │
  │ }                                                        │
  └─────────────────────────────────────────────────────────┘

  部署到云端时的差异：
    本地                       → 云端
    "file_name": "train.json"  → "file_name": "/root/data/train.json" (AutoDL)
                               → "file_name": "/content/data/train.json" (Colab)

  ★ 常见错误：
    - 把 system 写成 columns.system → 报错，必须用 tags.system_tag
    - 逗号/括号不匹配 → JSON 解析失败，llamafactory-cli 提示 "No dataset named..."
    - file_name 路径写错 → 数据集加载为空

  ★ 验证数据是否正确注册：
    训练启动后看日志第一行 "Num examples"，如果显示 ≥4000 说明正常；
    如果只有几百，说明 dataset_info.json 有问题或数据文件路径不对。

七、训练完如何推理？
══════════════════════════════════════════════════════════════

  【方案 A】本地推理（有 GPU）

  推理脚本：test_lora_batch.py（复制到本地修改路径即可）

  核心代码：
  ┌─────────────────────────────────────────────────────────┐
  │ import torch                                             │
  │ from transformers import AutoModelForCausalLM, AutoTokenizer │
  │ from peft import PeftModel                               │
  │                                                          │
  │ BASE_MODEL = "Qwen/Qwen2.5-3B-Instruct"   # 或 7B       │
  │ LORA_PATH = "./chat_style_lora_7b_v1"     # LoRA 权重目录│
  │                                                          │
  │ # 加载基座模型 + LoRA 权重                               │
  │ tokenizer = AutoTokenizer.from_pretrained(BASE_MODEL)    │
  │ model = AutoModelForCausalLM.from_pretrained(            │
  │     BASE_MODEL,                                          │
  │     torch_dtype=torch.bfloat16,    # 3B 用 float16       │
  │     device_map="auto"                                    │
  │ )                                                        │
  │ model = PeftModel.from_pretrained(model, LORA_PATH)      │
  │                                                          │
  │ # ★ 推理时用人格化 system prompt                         │
  │ SYSTEM_PROMPT = "你是阿狸，一个爱撒娇、喜欢黏着喵喵的女生。│
  │                 你正在和男朋友喵喵微信聊天。回复短小精炼，  │
  │                 爱撒娇。"                                 │
  │                                                          │
  │ def chat(user_input):                                    │
  │     text = (                                             │
  │         f"<|im_start|>system\n{SYSTEM_PROMPT}<|im_end|>\n"│
  │         f"<|im_start|>user\n{user_input}<|im_end|>\n"    │
  │         f"<|im_start|>assistant\n"                       │
  │     )                                                    │
  │     inputs = tokenizer(text, return_tensors="pt").to(model.device) │
  │     outputs = model.generate(                            │
  │         **inputs,                                        │
  │         max_new_tokens=128,                              │
  │         temperature=0.8,                                 │
  │         do_sample=True,                                  │
  │         top_p=0.9,                                       │
  │         pad_token_id=tokenizer.eos_token_id              │
  │     )                                                    │
  │     return tokenizer.decode(                             │
  │         outputs[0][len(inputs[0]):],                     │
  │         skip_special_tokens=True                         │
  │     ).strip()                                            │
  └─────────────────────────────────────────────────────────┘

  推理参数说明：
    temperature=0.8  → 较高温度增加多样性（聊天需要灵活，不需要确定性）
    top_p=0.9        → nucleus sampling
    max_new_tokens=128 → 限制回复长度（微信聊天通常短）

  ─────────────────────────────────────────────────────────

  【方案 B】Colab 推理（免费 T4）

  直接运行 colab_lora_train.ipynb Step 6，交互式对话。

  ─────────────────────────────────────────────────────────

  【方案 C】AutoDL 云端推理

  上传 test_lora_batch.py → 修改 BASE_MODEL/LORA_PATH → 运行：
    python test_lora_batch.py
  注意：AutoDL 终端输入中文会损坏 Unicode，用批量脚本避免交互式输入。

  ─────────────────────────────────────────────────────────

  ★ System Prompt 策略（核心经验）：
  ┌─────────────────────────────────────────────────────────┐
  │ 训练时（数据里）—— 泛用短 prompt                          │
  │   "你正在微信上与朋友聊天。"                              │
  │                                                          │
  │ 推理时（代码里）—— 人格化长 prompt                        │
  │   "你是阿狸，一个爱撒娇、喜欢黏着喵喵的女生。             │
  │    你正在和男朋友喵喵微信聊天。回复短小精炼，爱撒娇。"     │
  │                                                          │
  │ 原因：                                                    │
  │  - 长 prompt 在训练数据中重复 4592 次 → 模型死记硬背      │
  │  - 短 prompt 让模型从对话内容中自然学习风格               │
  │  - 推理时注入身份 → 且不影响训练质量                      │
  └─────────────────────────────────────────────────────────┘

十、常见问题排查
══════════════════════════════════════════════════════════════

  Q: 训练样本只有 900 条，不是 4592 条？
  A: LLaMA-Factory v0.9+ 要求 system 后第一条是 human。检查 build_lora_data.py
     是否有 first_me 过滤逻辑。重新构建数据再上传。

  Q: "Dropped invalid example: []" 大量出现？
  A: 同上，数据格式不满足 system→human→gpt 顺序。重新跑 build_lora_data.py。

  Q: bitsandbytes 找不到？
  A: AutoDL 无外网装不了。去掉 --quantization_bit 4，7B LoRA BF16 24GB 刚好。

  Q: HuggingFace 连不上？
  A: export HF_ENDPOINT=https://hf-mirror.com
     或用 ModelScope 下载模型到本地路径。

  Q: 模型输出 [sticker:xxx]？
  A: 原始聊天记录中的表情包占位符，训练数据自带，正常现象。

  Q: 模型分不清自己是谁？
  A: 训练用的 system prompt 太泛。推理时换人格化 prompt 即可。

  Q: AutoDL 终端输入中文后 tokenizer 崩溃？
  A: 终端删除字符产生异常空格。用批量测试脚本代替交互式输入。
