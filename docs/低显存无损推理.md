# **📝 Qwen2.5-7B 1080Ti 低显存无损推理部署指南**

本笔记记录了将微调后的 **Qwen2.5-7B-Instruct** 模型，在 **1080Ti (11G 显存)** 显卡上成功实现低显存（\< 6G）且不损失微调智商的完整踩坑与解决全过程。

## **🛠️ 环境依赖与准备**

在老旧显卡（Pascal 架构如 1080Ti）上跑量化推理，必须安装以下核心依赖：

1. **基础微调/推理框架**：LLaMA-Factory  
2. **动态量化后端**：bitsandbytes (建议版本 $\\ge$ 0.39.0)  
   * *安装命令（解决国内源下载假死问题）*  
   * 直接在本地电脑上下载whl 再拖拽传输进云端，云端本地安装：  
     Bash  
     pip cache purge

   pip install bitsandbytes-0.40.2-py3-none-any.whl

   * *环境验证*：  
     Bash

## **❌ 踩坑记录与排查过程**

### **问题 1：全精度推理导致显存炸裂 (OOM)**

* **现象**：直接运行常规 llamafactory-cli chat 加载 7B 原生 bf16 格式，显存占用超出 14G，只能在4090（24G）上运行推理，1080Ti (11G) 直接爆显存崩溃。  
* **初次尝试**：在命令中强行加入 \--quantization\_bit 4 进行动态量化。  
* **引发新问题**：显存占用虽然降到了 6G（在bash使用nvidia-smi查看命令时刻的显卡状态），但导致模型**拟合度严重下降**。

### **问题 2：LLaMA-Factory 导出量化模型 Bug 连环环**

* **目标**：尝试先合并（Merge）LoRA 再单独导出精细量化的 4-bit 模型（GPTQ/AWQ 或 GGUF 格式）以保住智商。  
* **报错 A**：ValueError: Please merge adapters before quantizing the model.  
  * *原因*：框架不支持在未合并时直接一步到位导出量化版。  
* **报错 B**：ValueError: Quantization dataset is necessary for exporting.  
  * *原因*：即便先合并了模型（生成了 qwen2.5\_7b\_merged 文件夹），但在执行第二步 export 压缩成 4-bit（或 GGUF）时，LLaMA-Factory 的数据加载器因路径解析 Bug 无法识别 \--dataset 参数，陷入强制要求校准数据集的死循环。

## **终极解决办法：“金蝉脱壳”两步法**

为了绕过 LLaMA-Factory 导出机制的底层 Bug，同时避免动态量化破坏 LoRA 权重，最终采用“高配卡合并全精度 $\\rightarrow$ 老显卡动态量化加载”的闭环方案。

### **步骤一：血肉融合（Merge 权重）**

首先，在有算力的环境下，将基础模型和 LoRA 插件融合成一个完整的 16 位大模型。此步骤彻底消除了 LoRA 独立存在的结构。

Bash  
llamafactory-cli export \\  
  \--model\_name\_or\_path /root/autodl-tmp/models/Qwen2.5-7B-Instruct \\  
  \--adapter\_name\_or\_path /root/autodl-tmp/output/chat\_style\_lora\_7b\_pure \\  
  \--template qwen \\  
  \--export\_dir /root/autodl-tmp/output/qwen2.5\_7b\_merged \\  
  \--export\_legacy\_format False

*注：此步骤成功后，原本的 LoRA 文件夹（如 chat\_style\_lora\_7b\_pure）可直接删除以节省硬盘空间。*

### **步骤二：无损动态 4-bit 启动推理**

将完全体模型（Merged）作为输入，利用 bitsandbytes 在启动的瞬间动态拧成 4-bit 读入显存。

Bash  
llamafactory-cli chat \\  
  \--model\_name\_or\_path /root/autodl-tmp/output/qwen2.5\_7b\_merged \\  
  \--template qwen \\  
  \--quantization\_bit 4
